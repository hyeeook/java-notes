새 파일을 만들 때 아래 구조와 컨벤션을 따른다.

3. [추가 섹션 제목](#3-추가-섹션-제목)

# 1. [개념]이 왜 필요한가?

```java

```
[문제점 설명. 두 줄 이상이면 <br>로 줄바꿈]

```
## 코드 블록 컨벤션
- 언어는 항상 `java` 명시
- 잘못된 방법: `// ❌` 주석으로 표시
- 올바른 방법: `// ✅` 주석으로 표시
- 생략된 구현: `{ ... }` 사용

## 문제-해결 패턴
```
문제: [상황 설명]
```java
// ❌ 잘못된 코드
```
[왜 문제인지 설명]

해결: [해결 방향]
```java
// ✅ 올바른 코드
```
[왜 해결되는지 설명]
```

## 비교표
```
| | [옵션 A] | [옵션 B] |
|:---:|:---:|:---:|
| [기준] | ✅ | ❌ |
```
- 열 정렬은 `:---:` (가운데 정렬) 사용

## 인용구(blockquote) 사용 기준
- 핵심 원칙, 설계 격언 (예: "구현이 아닌 추상에 의존하라")
- Effective Java 등 책 인용 (예: `Effective Java Item 18: "..."`)
- 다른 파일 참조 (예: `> 자세한 내용은 defensive-copy.md 참고.`)
- 보충 설명 (💡 팁, ⚠️ 주의, 🔍 심화)

## 단락 내 줄바꿈
- 같은 단락 안에서 줄바꿈이 필요한 경우 `<br>` 사용
- 새 단락은 빈 줄로 구분
```
